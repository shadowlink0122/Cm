## リリースノート形式

各リリースノートは以下の形式に従います:

```markdown
# Cm vX.Y.Z

リリース日: YYYY-MM-DD

## ハイライト
主要な変更点の概要

## 新機能
- 機能1
- 機能2

## 改善
- 改善1

## バグ修正
- #123: 修正内容

## 破壊的変更
- 変更内容

## 既知の問題
- 問題

## 謝辞
コントリビューターへの感謝
```

## バージョニング

[Semantic Versioning](https://semver.org/) に従います:

```
MAJOR.MINOR.PATCH

MAJOR: 互換性のない変更
MINOR: 後方互換な機能追加
PATCH: 後方互換なバグ修正
```
