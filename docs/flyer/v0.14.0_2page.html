Cm (シーマイナー) プログラミング言語 v0.14.1

OS開発からWebアプリまで、1つの言語でカバーするシステム言語

Cmとは？

モダンなC/C++を目指しました

基本構文はC/C++で、Rust/TypeScriptなどのモダンな言語機能を取り入れました。

ベアメタルプログラミングからWebアプリ開発まで、全てを1つの言語で行えるようにしたいというコンセプトです。

Cmが目指す3つの柱

  1. 実用性重視 - 理論より実践を優先
  2. 型安全性 - コンパイル時の厳密な型チェック
  3. 高速実行 - LLVM最適化でC++並みの速度

1. Interface/Impl + with自動実装

struct Point with Debug { int x; int y; } interface Printable { void print(); } impl Printable for Point { void print() { println("({self.x},{self.y})"); } } int main() { Point p = Point{10, 20}; p.debug(); // Point{x:10, y:20} p.print(); // (10,20) }

2. ジェネリクス + Match式

enum Result<T> { Ok(T), Err(string) } <T> T unwrap(Result<T> r) { match (r) { Result::Ok(v) => return v, Result::Err(e) => { println(e); exit(1); } } } int main() { Result<int> r = Result::Ok(42); println(unwrap(r)); // 42 }

3. 演算子オーバーロード (v0.14.0)

struct Vec2 { int x; int y; } impl Vec2 { operator Vec2 +(Vec2 o) { return Vec2{self.x+o.x, self.y+o.y}; } } int main() { Vec2 a = Vec2{1, 2}; Vec2 b = Vec2{3, 4}; Vec2 c = a + b; // Vec2{4, 6} a += b; // += は自動生成 }

4. インラインユニオン型 (v0.14.0)

int | null maybe_find(int[5] arr, int target) { for (int i = 0; i < 5; i++) { if (arr[i] == target) return arr[i]; } return null; }

サンプルプログラム

例1: FizzBuzz（Interface活用）

interface IFizzbuzz { void fizzbuzz(); } impl IFizzbuzz for int { void fizzbuzz() { if (self % 15 == 0) println("FizzBuzz"); else if (self % 3 == 0) println("Fizz"); else if (self % 5 == 0) println("Buzz"); else println(self); } } int main() { for (int i = 1; i <= 100; i++) { i.fizzbuzz(); } }
ポイント: プリミティブ型（int）にメソッド追加可能

コンパイラアーキテクチャ

多段IR変換パイプライン:

Source → AST → HIR → MIR → LLVM IR → Native
  • AST: 構文解析結果（型推論前）
  • HIR: 高レベルIR（型解決・脱糖後）
  • MIR: 制御フローグラフ（最適化対応）

MIR最適化パイプライン (独自実装)

  • 定数畳み込み - コンパイル時計算
  • DCE - 到達不能コード除去
  • SCCP - 疎条件付き定数伝播
  • CSE - 共通部分式除去
  • インライン展開 - 小関数の埋め込み
  • 末尾呼出最適化 - 再帰をループに変換

+ LLVM O3最適化（ベクトル化、ループ最適化等）

🖥️ UEFI開発

UEFIアプリケーション

CmコンパイラでUEFIアプリケーションを直接コンパイル可能。 QEMUでの動作を確認済みです。

import uefi::text::{uefi_println, uefi_clear_screen}; import uefi::mem::uefi_stall; ulong efi_main(void* image_handle, void* system_table) { uefi_clear_screen(system_table); uefi_println(system_table, "Hello from Cm!"); uefi_stall(system_table, 5000000); return 0; // EFI_SUCCESS }

ビルド手順

# UEFIオブジェクトにコンパイル cm compile --target=uefi -o hello.o hello.cm # EFIバイナリにリンク lld-link /subsystem:efi_application \ /entry:efi_main /out:BOOTX64.EFI hello.o # QEMUで実行テスト qemu-system-x86_64 \ -drive if=pflash,format=raw,file=OVMF.fd \ -drive format=raw,file=fat:rw:esp \ -net none -nographic

⚙️ ベアメタル開発

OS依存機能を排除したベアメタルターゲットで、 組み込みシステムやカスタムOS向けのコードをコンパイルできます。

// ベアメタル: OS依存なしでコンパイル可能 struct Register { int* base; int offset; } int main() { int[4] mmio = [0, 0, 0, 0]; Register r = Register{mmio as int*, 2}; *(r.base + r.offset) = 0xFF; return *(r.base + r.offset); }

🌐 Web開発 (JS/WASMバックエンド)

JavaScriptへのコンパイル

CmコードをJavaScriptに変換し、Node.jsやブラウザで実行できます。 v0.14.0でテスト通過率が55%→85%に大幅向上。

int main() { println("Hello from Cm → JS!"); int[5] nums = [1, 2, 3, 4, 5]; int sum = nums.reduce(0, (a,b) => a+b); println("sum = {sum}"); // 15 }
$ cm compile --target=js hello.cm -o hello.js $ node hello.js Hello from Cm → JS! sum = 15

WebAssemblyへのコンパイル

$ cm compile --target=wasm hello.cm -o hello.wasm $ wasmtime hello.wasm Hello from WASM!

作ってみた感想

このプロジェクトを始めるまで、プログラミング言語を作った経験はなく、低レイヤーのデバッグ知識もありませんでした。前身のCb（シーフラット）は細部を詰めずに「とりあえず動くもの」を作っていましたが、設計やゴールが曖昧だったため一度挫折。逆に「設計さえしっかりすれば次はうまくいくはず」と考え、再出発してCm（シーマイナー）と名付けました。

目指していたコンパイラにはある程度近づきましたが、納得していない点も多いです。モダンな言語の良いところとして、変数宣言はRustやTSのように後置型注釈のほうが見やすかったり、func/fnキーワードを使ったほうがBNFとしての構文や意味解析がシンプルになったかもしれないと思っています。

テストはある程度行なっておりCIも導入しましたが、実際のユースケースに合わせた改善やバグ修正を今後も行なっていこうと思います。

ディレクトリ構成・言語設計・中間表現・最適化などを生成AIと壁打ちしながら設計の手がかりを得て、今に至ります。設計に関する過去のドキュメントは全てdocs/archiveにまとまっているので興味があればぜひ見てみてください！

※本プロジェクトはバイブコーディングで作られており、作者は肝心な低レイヤーのデバッグ手法などをあまり理解できていない点が残念ポイントになっています！
