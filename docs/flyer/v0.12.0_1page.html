Cmとは？

名前の由来: C → C++ → C# と+が増えていくので逆の命名に。機能的にはC++のモダン版を目指しています。
📢 Cbからの変更点:
  • 言語名変更: Cb → Cm（シーマイナー）
  • コンパイラ化: インタープリタ→LLVM JIT
  • マルチバックエンド: Native/WASM/JS対応
  • 本格的ジェネリクス: 型制約・モノモーフィゼーション

なぜCmを作ったのか？

「使いやすく、安全で、高速な言語」を作りたかった

OS開発からWebアプリまで、1つの言語で全てをカバー。低レベルな制御とモダンな書き方を両立。

Cmが目指す3つの柱

  1. 実用性重視 - 理論より実践を優先
  2. 型安全性 - コンパイル時の厳密な型チェック
  3. 高速実行 - LLVM最適化でC++並みの速度

Cmの特徴

🚀 Cmの強み:
  • C++並みの速度: LLVMネイティブコンパイル
  • モダン構文: Rust/TS風の直感的な書き方
  • マルチターゲット: 1ソースで複数環境に出力

1. Rust風Interface/Impl + with自動実装

// with Debug: 自動でdebug()メソッド生成 struct Point with Debug { int x; int y; } // 明示的なimpl interface Printable { void print(); } impl Printable for Point { void print() { println("({self.x},{self.y})"); } } int main() { Point p = Point{10, 20}; p.debug(); // Point{x:10, y:20} p.print(); // (10,20) }

2. ジェネリクス + Match式

enum OptionTag { Some, None } struct Option<T> { OptionTag tag; T value; } <T> Option<T> some(T v) { return Option<T>{OptionTag::Some, v}; } int main() { Option<int> opt = some(42); match (opt.tag) { OptionTag::Some => println(opt.value), _ => println("None") } }

3. ポインタ・配列（C++互換）

int[10] arr; for (int i = 0; i < 10; i++) { arr[i] = i * 2; } int* p = &arr[0]; println(*p); // 0

サンプルプログラム

例1: FizzBuzz（Interface活用）

interface IFizzbuzz { void fizzbuzz(); } impl IFizzbuzz for int { void fizzbuzz() { if (self % 15 == 0) println("FizzBuzz"); else if (self % 3 == 0) println("Fizz"); else if (self % 5 == 0) println("Buzz"); else println(self); } } int main() { for (int i = 1; i <= 100; i++) { i.fizzbuzz(); } }
ポイント: プリミティブ型（int）にメソッド追加可能
