## 設計原則

1. **明確な責務分離**: 各コンパイルフェーズは独立して動作
2. **テスト容易性**: 各フェーズを単体でテスト可能
3. **拡張性**: 新しいバックエンドや最適化パスの追加が容易
4. **良いエラーメッセージ**: ソース位置情報の保持
