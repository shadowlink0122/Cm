# PR作成・レビュールール

## 言語
- すべて**日本語**で記述

## PRタイトル形式
```
<種類>: <簡潔な説明>

種類: feat, fix, docs, refactor, test, chore
```

## PR作成時
1. テンプレートに従って本文を記入
2. 関連Issueをリンク
3. チェックリストを確認
4. `make format` でC++/Cmコードをフォーマット

## レビュー時
1. 設計ドキュメント (`docs/design/`) との整合性確認
2. クロスバックエンド互換性確認
3. テストの有無確認
   - ファイル名に `test_` プレフィックスがないこと
   - `/tmp` へのデータファイル書き込みがないこと（`data/` を使用）
4. **⚠️ ローカルパス情報の確認（重要！）**
   - `/Users/` や `/home/` 等のローカルパスが含まれていないこと
   - すべてのドキュメントリンクはプロジェクトルートからの相対パスであること
   - 確認コマンド: `grep -rn "/Users/" docs/ .agent/ --include="*.md"`
5. 日本語でコメント

## マージ条件
- CI全パス
- 1人以上のApprove
- コンフリクトなし
