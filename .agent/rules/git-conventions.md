# コミット・ブランチルール

## ブランチ命名規則
```
feature/v<version>      - 機能開発ブランチ
fix/<issue-number>      - バグ修正
hotfix/<description>    - 緊急修正
release/v<version>      - リリース準備
```

## コミットメッセージ
- **日本語**で記述
- 1行目: 変更の要約（50文字以内）
- 空行
- 本文: 詳細説明（必要な場合）

### コミット種類プレフィックス
```
feat:     新機能
fix:      バグ修正
docs:     ドキュメント
refactor: リファクタリング
test:     テスト追加・修正
chore:    ビルド・設定変更
perf:     パフォーマンス改善
```

## コミット前チェック
1. `make format` - C++/Cmコードのフォーマット
2. `make build` - ビルド確認
3. `make tip` or `make tlp` - テスト確認
4. 不要な変更が含まれていないか確認

## プッシュ前チェック
- リモートの最新を取得
- コンフリクトがないことを確認
