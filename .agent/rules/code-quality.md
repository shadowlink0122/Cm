# コード品質ルール

## コードスタイル

### C++ソースコード
- **C++20標準**を使用
- 4スペースインデント
- ブレース: K&Rスタイル
- 行長: 120文字以内
- ヘッダーガード: `#pragma once`

### Cmソースコード
- 4スペースインデント
- 関数名: snake_case
- 型名: PascalCase
- 定数: UPPER_SNAKE_CASE

## コメント規約
- **設計意図**を書く（何をしているかではなく、なぜそうするか）
- TODOコメント形式: `// TODO(担当者): 説明`
- 一時的なデバッグコードは残さない

## ファイル構成
- 1ファイル1000行目安
- 関連機能は同一ディレクトリに配置
- テストファイルは`tests/`以下に配置

## 禁止事項
- `using namespace std;`（ヘッダーファイル内）
- マジックナンバー（定数定義を使用）
- グローバル変数（必要な場合はシングルトンパターン）
- 未使用のinclude

## 推奨事項
- constを積極的に使用
- auto型推論は明確な場合のみ
- 範囲forループを優先
- スマートポインタを使用
